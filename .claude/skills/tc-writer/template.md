# testcase.md 템플릿

## 사용법
아래 템플릿을 복사해 `output/TC-{NNN}_{Service}_{Feature}/testcase.md`로 저장.

---

```markdown
# TC-{NNN}: {제목}

## 전제조건
- {브라우저/환경 조건}
- {네트워크 조건}
- {계정/로그인 조건 (필요 시)}

## 테스트 단계
| 단계 | 액션 | 기대 결과 |
|------|------|-----------|
| 1 | {URL} 접속 | 페이지 로드 완료 |
| 2 | {요소} 클릭 | {결과} |
| 3 | {입력/액션} | {결과} |

## 기대 결과
- ✅ ...
- ✅ ...

## 실패 조건
- ❌ 페이지 로드 실패
- ❌ {요소} 없음 또는 클릭 무반응
- ❌ ...

## 알려진 제약사항
<!-- domain-skills/{domain}/scraping.md에서 확인한 내용 반영 -->
- (없으면 삭제)
```

---

## 작성 원칙

- **사용자 관점** 기술 — "버튼을 클릭한다" (O), "locator('#btn').click()" (X)
- **한 TC = 하나의 흐름** — 여러 흐름은 TC 분리
- **실패 조건 필수** — 테스트 종료 판단의 근거
- **제약사항 명시** — 외부 앱, 로그인, 지역 제한 등
